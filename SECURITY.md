# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AI-Bastion, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email: Open a private issue on this repository or contact the maintainers through GitHub.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested fix (if you have one)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Assessment:** Within 1 week
- **Fix:** As soon as possible, depending on severity

## Scope

This policy covers the AI-Bastion repository code and documentation. It does not cover:
- The AI agents themselves (OpenClaw, etc.)
- Third-party packages installed by AI-Bastion (nftables, stubby, auditd)
- Your specific deployment configuration
